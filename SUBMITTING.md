# 提交代码指南

本项目使用 Git 工作流管理代码。提交代码的一般步骤如下：

1. 查看当前修改：
   - `git status` 查看工作区与暂存区的文件变化。
   - 如需确认具体改动，可用 `git diff` 查看差异。

2. 将改动加入暂存区：
   - 对所有改动：`git add .`
   - 仅添加指定文件：`git add <文件路径>`

3. 提交改动：
   - 使用清晰的提交信息：`git commit -m "描述本次改动"`
   - 提交前可通过 `git diff --cached` 再次确认暂存区内容。

4. 推送到远程仓库：
   - 先拉取最新更新避免冲突：`git pull`
   - 推送当前分支：`git push`（若是首次推送新分支，可使用 `git push -u origin <分支名>`）

5. 创建合并请求 / Pull Request（如适用）：
   - 在代码托管平台新建 PR，填写改动摘要与测试情况。
   - 根据代码审核意见进行必要的更新后重新提交。

常见问题：
- 如果提交时提示无改动，请确保已使用 `git add` 将文件加入暂存区。
- 如果遇到合并冲突，先使用 `git status` 定位冲突文件，手动解决后 `git add` 并重新 `git commit`。
